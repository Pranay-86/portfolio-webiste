State Estimation of a Mobile Robot using Extended Kalman Filter under Model Bias and Encoder Bias

April 2026

Investigates the robustness of EKF-based localisation on a low-cost mobile robot when the motion model and encoder measurements contain systematic errors. It comprises two complementary parts: (1) experimental characterisation of EKF robustness under systematic encoder bias, and (2) analysis of EKF performance under velocity model bias — together forming a comprehensive study of how model inaccuracies affect state estimation.

Problem Statement

Mobile robots must continuously estimate their position while moving. Indoors, they rely on onboard sensors (wheel encoders, IMU) and motion models — but real robotic systems suffer from calibration inaccuracies, wheel slip, and measurement bias, which can cause localisation algorithms to accumulate error or become unstable. This project investigates how such systematic errors affect EKF-based localisation.

  • Sensor noise and measurement bias
  • Accumulation of estimation error during motion
  • Stability under incorrect measurements
  • Non-linear relationship between bias magnitude and estimation error

Hardware Platform

  • Microcontroller: ESP32
  • Sensors: Wheel encoders, MPU6050 IMU, HC-SR04 ultrasonic distance sensor
  • Actuation: 2 DC motors with TB6612FNG motor driver
  • Data logging: MicroSD module

System Model

State vector: x = [position_x, position_y, orientation_theta] Prediction (motion model): x = x + v·cos(theta)·dt y = y + v·sin(theta)·dt theta = theta + omega·dt (v: linear velocity from PWM + calibration constant; omega: gyro angular velocity) Measurement model: z = d0 − x (distance to wall, from ultrasonic sensor)

Bias introduction (two types):

  • Encoder bias: injected into the velocity model as v = k·PWM·(1 + bias), tested at 0%, 2%, 5%, 10%, 15%, 20%, 25%, 30%
  • Model bias: similar velocity scaling, tested at 0%, 5%, 10%, 15%, 20%

Methodology

  • Assemble the 2WD robot platform with ESP32, motors, encoders, IMU, ultrasonic sensor
  • Interface encoders, IMU, and ultrasonic sensor for data acquisition
  • Baseline testing — drive the robot along straight-line trajectories to record raw sensor data
  • Implement the EKF on the ESP32 to estimate pose (x, y, theta) using encoder velocity and IMU angular rate, with ultrasonic distance as the measurement
  • Log sensor measurements and EKF estimates to MicroSD for offline analysis
  • Inject controlled systematic bias into the velocity model (encoder-based and generic model bias)
  • Run repeated trials at each bias level along the same trajectory
  • Analyse results in Python — trajectory plots, error metrics, uncertainty evolution

Key Findings

1. Position tracking

At low bias (0%–10%) the EKF-estimated position closely follows the true path. At higher bias (15%–20%), deviations become noticeable but the filter does not diverge.

2. Estimation error vs. bias (non-monotonic behaviour)
Bias levelFinal estimation error

Error does not increase linearly with bias — the filter adapts by shifting trust from the model to sensor measurements, resulting in better accuracy at higher bias when sensor noise is low.

3. Bounded error

The EKF error remains bounded and self-corrects after measurement spikes, demonstrating robustness.

4. Uncertainty convergence

Covariance Pxx decreases and stabilises for all bias levels, indicating the filter gains confidence over time.

5. Robustness evaluation

The EKF remains stable despite model errors up to 20% bias, dynamically balancing prediction and measurement so estimation does not diverge.

Performance Metrics

  • Position accuracy: RMSE between estimated and true positions
  • Error evolution: error-vs-time plots to assess stability
  • Final estimation error: error at the end of the run, per bias level
  • Uncertainty (Pxx): covariance evolution over time

Interpretation

The non-monotonic error behaviour is explained by the adaptive nature of the EKF: as the model becomes less reliable, the filter automatically gives more weight to sensor measurements. When sensor noise is relatively low, this increased reliance on measurements can actually improve accuracy — leading to lower error at higher bias.

Statistical Overconfidence Precedes Accuracy Degradation in the Unscented Kalman Filter Under Process-Noise Underestimation

March 2026

Examines the behaviour of the Unscented Kalman Filter under systematic process-noise covariance underestimation — specifically the relationship between numerical accuracy and statistical consistency.

Background

The UKF relies on a fundamental assumption: process and measurement noise covariances must be correctly specified. In practice these covariances are rarely known precisely, and incorrect specification can significantly alter filter behaviour, leading to overconfidence.

Problem Statement

Most practical filter evaluations rely on trajectory plots or RMSE, which only measure numerical accuracy and do not assess whether reported uncertainty is statistically valid. This project investigates the decoupling between estimation accuracy and statistical consistency.

System Model

State vector: x_k = [x_k, y_k, v_k, ψ_k]ᵀ (planar position, velocity, heading angle) Process dynamics: x_(k+1) = x_k + v_k·cos(ψ_k)·Δt y_(k+1) = y_k + v_k·sin(ψ_k)·Δt v_(k+1) = v_k + w_v,k ψ_(k+1) = ψ_k + w_ψ,k Process noise mis-specification: Q_filter = γ · Q_true, γ ∈ {1, 1/2, 1/3, 1/5, 1/9, 1/15}

Performance Metrics

  • RMSE = √( (1/T)·Σ‖x_k − x̂k|k‖² ) — measures numerical accuracy only
  • NEES (Normalized Estimation Error Squared): ε_k = (x_k − x̂k|k)ᵀ·P⁻¹k|k·(x_k − x̂k|k). For a consistent estimator, E[ε_k] = n (state dimension) — NEES assesses whether reported covariance matches true estimation error

Key Findings

  • Clear decoupling: accuracy and consistency degrade at different rates under process-noise underestimation
  • NEES analysis: correct specification keeps NEES near the theoretical expectation (n=4); underestimation causes NEES to rise rapidly, exceeding 95% chi-squared confidence bounds. At γ = 1/15, mean NEES reaches ~30.5 — nearly an order of magnitude above baseline
  • RMSE analysis: baseline ~0.56 m; at γ = 1/15, RMSE reaches ~0.81 m (<50% increase) — no catastrophic failure in accuracy

  • NEES grows by a factor of ~7.6 across the mismatch range

  • Covariance inflation alone does not restore consistency

Methodology

  • Monte Carlo simulations: 200 independent runs per configuration
  • Implementation: Python with NumPy, SciPy, Matplotlib
  • Key observation: accuracy-only evaluation is insufficient for certifying filter reliability

Future Scope

CRISPR Off-Target Prediction using Multi-Modal Deep Learning

March 2026

A multi-modal deep learning framework for CRISPR-Cas9 off-target prediction, integrating three complementary modalities: DNA sequence context, engineered auxiliary biological features, and epigenetic signals.

Background

CRISPR-Cas9 is a powerful genome editing tool, but off-target effects remain a critical concern. Traditional computational approaches rely primarily on sequence similarity metrics and fail to capture the broader biochemical and epigenetic context governing Cas9 activity.

Problem Formulation

Given a guide RNA sequence (sgRNA) and a candidate genomic DNA site, predict the probability of an unintended double-strand break: y = σ(f_θ(x_seq, x_aux, x_epi))

Model Architecture — Three Parallel Branches

  • Sequence encoder (DNABERT-2): processes a 23-mer sgRNA–DNA pair through 12 Transformer layers with multi-head self-attention; CLS token yields a 768-dimensional embedding
  • Auxiliary feature network: 15-dimensional biological feature vector through a 2-layer MLP (128 → 64) with ReLU — features include mismatch positions, GC content, PAM characteristics
  • Epigenetic feature network: 8-dimensional DNase-derived features through a 2-layer MLP (64 → 32) with ReLU — features include raw DNase signal, accessibility statistics, GC interaction

Fusion & classification: concatenation into an 864-dimensional joint representation, followed by a 2-layer classification head with dropout (0.3) and a sigmoid output.

Datasets

Integrated datasets (500K+ samples): DeepCRISPR (GUIDE-seq, HTGTS, CIRCLE-seq), DeepHF (WT, ESP, HF1 variants), SITE-Seq, Wang (LAM-PCR), Klein (primary human T-cells), Designer (high-specificity guides).

Training subset: 80K high-quality samples.

Enhanced Quantum-Classical Hybrid Model for Predicting Chronic Heart Disease in Healthcare 4.0

April 2026

An Enhanced Quantum-Classical Hybrid Prediction Framework that benchmarks classical machine learning models against quantum kernel methods on the UCI Cleveland Heart Disease dataset, further validated on the Statlog Heart dataset.

Problem Identification Factors

PIF-1 — The quantum encoding bottleneck: standard quantum encoding scales O(N) qubits; 13 features impose intractable qubit overhead; a direct ZZFeatureMap on selected features achieves only 76.37% (below the classical baseline of 83.54%).

PIF-2 — Poor feature separability in raw clinical data: raw clinical features exhibit non-linear interactions; fixed angle encoding creates noisy, poorly separable quantum states; statistical feature selection loses inter-feature interaction information.

PIF-3 — NISQ hardware noise degrading kernel fidelity: quantum kernel computation is sensitive to depolarising errors; a 1% depolarising error model degrades accuracy substantially; Zero Noise Extrapolation (ZNE) addresses this through noise scaling.

Datasets

  • UCI Cleveland Heart Disease: 303 patient records (297 after cleaning); 13 clinical features + binary target; 160 negative (53.9%), 137 positive (46.1%) — near-balanced binary classification
  • Statlog Heart Dataset: 270 patient records; 13 clinical features aligned with Cleveland; 150 negative (55.6%), 120 positive (44.4%) — a different clinical cohort with distributional shift
  • Mega-dataset: combined Cleveland + Statlog, 567 patient records; 453 training / 114 test (80/20 stratified split)

Methodology — Nine-Phase Pipeline

Phase 1 — Data preparation: standardisation, binarisation, feature selection (ANOVA F-score, k=6).

Phase 2 — Classical baseline models: Logistic Regression (82.69%), Decision Tree (75.94%), Random Forest (81.85%), AdaBoost (81.84%), SVM+RBF (82.28%), SVM+PCA (83.54% — best), Voting Ensemble (81.86%).

Phase 3 — Basic QSVM: ZZFeatureMap (reps=2, linear entanglement), 6 quantum-scaled features, 76.37% CV accuracy.

Phase 4 — Advanced QSVM + ZNE: full entanglement (15 qubit pairs), Mitiq Zero Noise Extrapolation, 90.00% on a 20-patient subset (proof-of-concept).

Phase 5 — Hybrid NQE+QSVC (proposed): Neural Quantum Embedding network (13 → 10 → 6) with ReLU + Tanh activation; 89.87% CV accuracy, 86.67% test accuracy — a 6.33 percentage-point improvement over the classical baseline.
